# Прошивка RoverCtrl v2.0 — AP Mode

## Обзор изменений

### Новая сетевая архитектура
**Раньше:** Оба модуля подключались к внешнему роутеру (DHCP, резерв IP).
**Теперь:** Ровер создаёт WiFi AP, турель и телефон подключаются к нему.

```
┌─────────────────┐
│  Ровер ESP32S3  │  ← AP mode, 192.168.4.1
│  (rover_ap.ino) │
└────────┬────────┘
         │ WiFi "RoverAP"
    ┌────┴────┐
    │         │
    ▼         ▼
┌───────┐  ┌───────┐
│Турель │  │Телефон│
│.4.2   │  │DHCP   │
└───────┘  └───────┘
```

### Преимущества AP mode
- Не нужен внешний роутер
- Работает в полевых условиях
- Фиксированные IP, не меняются
- Быстрое подключение (~2 сек)

## Протокол: телефон → ровер (UDP :4210)

```
SPD:{};STR:{};FWD:{};LASER:{}\n
```

| Поле | Диапазон | Как использует прошивка |
|------|----------|------------------------|
| SPD | -100..100 | **Игнорируется** (legacy) |
| STR | -100..100 | `map(-100,100, 40,140)` → серво руления |
| FWD | -100..100 | `abs(FWD)` → мощность, `sign(FWD)` → направление |
| LASER | 0 / 1 | GPIO17 вкл/выкл |

**ИСПРАВЛЕНО:** STR маппинг теперь `40..140` (было указано неверно как `145..35`).

## Протокол: телефон → Xiao Sense (UDP :4210)

```
PAN:{};TILT:{}\n
```

Android внутренне использует -100..+100, CommandSender масштабирует в -90..+90.

| Поле | Диапазон (UDP) | Маппинг прошивки |
|------|----------------|------------------|
| PAN | -90..+90 | `map(-90,90, 180,0)` → серво **ИНВЕРТИРОВАН** |
| TILT | -90..+90 | `map(-90,90, 0,180)` → серво прямой |

**PAN инвертирован**: +PAN = поворот вправо (уменьшение серво-градусов).

## Телеметрия: ровер → телефон (UDP :4211)

Отправляется каждые 500мс:

```json
{
  "bat": 100,
  "yaw": 0.0,
  "spd": 75,
  "pit": 0.0,
  "rol": 0.0,
  "rssi": -45,
  "rpmL": 120.5,
  "rpmR": 118.3
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| bat | int | Батарея % (TODO: реальное измерение) |
| yaw | float | Не реализовано (0.0) |
| spd | int | Последний `abs(FWD)` из команды |
| pit, rol | float | Не реализовано (0.0) |
| rssi | int | WiFi RSSI (dBm) |
| rpmL | float | Обороты левого колеса (PCNT энкодер) |
| rpmR | float | Обороты правого колеса (PCNT энкодер) |

## Motor Watchdog (НОВОЕ)

**Проблема:** Если телефон теряет связь, ровер продолжал ехать с последней командой.

**Решение:** Watchdog таймер 500ms. Если нет UDP пакетов дольше 500ms:
- Моторы останавливаются
- Лазер выключается
- Сообщение в Serial Monitor

```cpp
#define MOTOR_WATCHDOG_MS  500

void checkWatchdog() {
    if (lastCmdTime > 0 && (millis() - lastCmdTime) > MOTOR_WATCHDOG_MS) {
        if (lastFwd != 0) {
            Serial.println("WATCHDOG: stopping motors!");
            stopMotors();
        }
    }
}
```

## Аппаратная конфигурация

### Ровер (192.168.4.1) — rover_ap.ino

| Компонент | Пины |
|-----------|------|
| Мотор IN1/IN2 | 6, 7 |
| PWM L/R | 4, 5 |
| Энкодер L (A/B) | 12, 11 |
| Энкодер R (A/B) | 38, 37 |
| Серво руля | 13 |
| Лазер | 17 |

Энкодеры: **440 пульсов/оборот**, квадратурный счёт через PCNT.

### Xiao Sense турель (192.168.4.2) — turret_client.ino

| Компонент | Пин |
|-----------|-----|
| Серво Tilt | D2 (GPIO 2) |
| Серво Pan | D3 (GPIO 4) |
| Камера OV2640 | Встроенная |

## Прошивка

### Требования
- Arduino IDE 2.x
- ESP32 Board Package (esp32 by Espressif)
- Библиотеки: ESP32Servo

### Ровер
1. Открыть `firmware/rover_ap/rover_ap.ino`
2. Выбрать плату: **XIAO ESP32S3**
3. Upload

### Турель
1. Открыть `firmware/turret_client/turret_client.ino`
2. Выбрать плату: **XIAO ESP32S3 Sense** (или XIAO ESP32S3)
3. Upload

## Миграция с v1.0

1. Перешейте **оба** модуля новыми прошивками
2. Подключите телефон к WiFi "RoverAP" (не к домашней сети)
3. В приложении нажмите "Reset" для сброса IP на дефолтные
4. Нажмите "Connect"

## Известные ограничения

- Максимум ~3-4 WiFi клиента на AP (ESP32 ограничение)
- Дальность WiFi: ~30-50м на открытом пространстве
- Нет шифрования UDP трафика
