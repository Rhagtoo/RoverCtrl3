# Прошивка турели — Station Mode (v2.5)

## Описание

Прошивка для XIAO ESP32S3 Sense. Подключается к WiFi AP ровера,
управляет pan/tilt серво и транслирует MJPEG с камеры OV2640.

## Особенности

- **WiFi Station Mode** — подключается к "RoverAP", статический IP 192.168.4.2
- **Auto Reconnect** — переподключение каждые 5с при потере связи
- **MJPEG Stream** — HTTP стрим с OV2640 на порт 81
- **UDP Control** — приём команд pan/tilt на порт 4210
- **Dual Core** — UDP на Core 0, серво обновление на Core 1 (loop)
- **CR Tilt Servo** — позиционный контроллер с постоянной скоростью (ведёт себя как позиционное серво)
- **Watchdog 2с** — нет команд → фиксация текущей позиции tilt

## Серво

### PAN — позиционное серво

| Параметр | Значение |
|----------|----------|
| GPIO | 2 (D2) |
| Диапазон | 0°..180° |
| Маппинг | cmd -90..+90 → servo 180..0 (**инвертирован**) |

### TILT — Continuous Rotation серво с виртуальным углом

Tilt серво — CR (continuous rotation), управляется как позиционное через
позиционный контроллер с постоянной скоростью.

| Параметр | Значение | Описание |
|----------|----------|----------|
| GPIO | 4 (D4) | |
| TILT_NEUTRAL | 90 | PWM = стоп (подстроить 88-92) |
| TILT_DEADBAND | ±3° | Зона нечувствительности |
| TILT_MAX_SPEED | 50 | Макс отклонение PWM от neutral |
| TILT_DEG_PER_SEC_UP | 60° | Скорость вверх (против гравитации) |
| TILT_DEG_PER_SEC_DN | 90° | Скорость вниз (по гравитации) |
| TILT_MIN_ANGLE | 10° | Нижний предел виртуального угла |
| TILT_MAX_ANGLE | 170° | Верхний предел виртуального угла |

**Алгоритм:**
1. Команда → `tiltTargetAngle`
2. `error = target - virtualAngle`
3. Если `|error| < DEADBAND` → servo = NEUTRAL (стоп)
4. Иначе: `speed = sign(error) × TILT_MAX_SPEED` (постоянная скорость)
5. `PWM = NEUTRAL + speed`
6. Open-loop интеграция: `virtualAngle += actualDegPerSec × dt` (разные скорости для вверх/вниз)
7. Watchdog: нет команд 2с → target = текущий виртуальный угол (стоп)

| Параметр | Значение |
|----------|----------|

## Протокол

### Входящие команды (UDP :4210)

```
PAN:{val};TILT:{val}\n
```

| Поле | Диапазон | Маппинг |
|------|----------|---------|
| PAN | -90..90 | map(-90..90 → 180..0) — **инвертирован** |
| TILT | -90..90 | map(-90..90 → 0..180) → виртуальный target angle |

Android отправляет ±100, CommandSender масштабирует в ±90.

### HTTP endpoints (порт 81)

| Endpoint | Описание |
|----------|----------|
| `/stream` | MJPEG стрим (multipart/x-mixed-replace) |

## Установка

1. Открыть `turret_client.ino` в Arduino IDE
2. Выбрать плату: **XIAO ESP32S3** (или XIAO ESP32S3 Sense)
3. Установить библиотеку **ESP32Servo**
4. Upload

## Порядок запуска

1. Сначала включить ровер (AP должен быть доступен)
2. Затем включить турель (подключится к AP)
3. Подключить телефон к "RoverAP"

## Калибровка

Если tilt серво ползёт в покое — подстрой `TILT_NEUTRAL`:
```cpp
#define TILT_NEUTRAL 90   // попробуй 88, 89, 91, 92
```

Если скорость tilt неточная — замерь реальную скорость и подстрой:
```cpp
#define TILT_DEG_PER_SEC 90.0f  // °/сек при макс PWM отклонении
```

## Отладка

Serial Monitor (115200 baud) показывает:
- Параметры CR tilt при старте
- Статус WiFi подключения
- Ошибки камеры
