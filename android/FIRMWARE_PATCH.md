# Прошивка ровера — XIAO_ESP32S3_Motors_Laser_Servo_NEW_PCNT

## Текущее состояние

Прошивка **уже включает** энкодеры (PCNT) и передаёт `rpmL`/`rpmR` в телеметрии.
OdometryTracker работает в точном режиме из коробки.

## Протокол: телефон → ровер (UDP :4210)

```
SPD:{};STR:{};FWD:{};LASER:{}\n
```

| Поле | Диапазон | Как использует прошивка |
|------|----------|------------------------|
| SPD | -100..100 | **Игнорируется** |
| STR | -100..100 | `map(-100..100 → 145..35)` → серво руления |
| FWD | -100..100 | `abs(FWD)` → мощность мотора, `sign(FWD)` → направление |
| LASER | 0 / 1 | GPIO17 вкл/выкл |

## Протокол: телефон → Xiao Sense (UDP :4210)

```
PAN:{};TILT:{}\n
```

Android внутренне использует -100..+100, CommandSender масштабирует в -90..+90.

| Поле | Диапазон (UDP) | Маппинг прошивки |
|------|----------------|------------------|
| PAN | -90..+90 | `constrain(-90,90)` → `map(-90,90, 180,0)` → серво **ИНВЕРТИРОВАН** |
| TILT | -90..+90 | `constrain(-90,90)` → `map(-90,90, 0,180)` → серво прямой |

**PAN инвертирован**: положительная команда → уменьшение серво-градусов (180→0).
Это сделано для естественного направления: +PAN = поворот вправо (по конвенции).

## Телеметрия: ровер → телефон (UDP :4211)

Отправляется каждые 500мс:

```json
{
  "bat": 100,
  "yaw": 0.0,
  "spd": 75,
  "pit": 0.0,
  "rol": 0.0,
  "rssi": -45,
  "rpmL": 120.5,
  "rpmR": 118.3
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| bat | int | Батарея % (захардкожено 100) |
| yaw | float | Не реализовано (0.0) |
| spd | int | Последний `abs(FWD)` из команды |
| pit, rol | float | Не реализовано (0.0) |
| rssi | int | WiFi RSSI (dBm) |
| rpmL | float | Обороты левого колеса (PCNT энкодер) |
| rpmR | float | Обороты правого колеса (PCNT энкодер) |

## MJPEG стрим: Xiao Sense → телефон (HTTP :81)

Камера OV2640 на XIAO Sense стримит MJPEG по HTTP.
Ось камеры совпадает с лучом лазера.

| Endpoint | Описание |
|----------|----------|
| `http://192.168.88.23:81/stream` | MJPEG стрим (multipart/x-mixed-replace) |
| `http://192.168.88.23/capture` | Одиночный JPEG кадр |
| `http://192.168.88.23/status` | JSON: pan, tilt, heap |

Android подключается к `/stream` через `MjpegDecoder` для PiP.

## Аппаратная конфигурация

### Ровер (192.168.88.24) — rover_esp32s3.ino

| Компонент | Пины |
|-----------|------|
| Мотор IN1/IN2 | 6, 7 |
| PWM L/R | 4, 5 |
| Энкодер L (A/B) | **12, 11** |
| Энкодер R (A/B) | **38, 37** |
| Серво руля | 13 |
| Лазер | 17 |

Энкодеры: **440 пульсов/оборот**, квадратурный счёт через PCNT (аппаратный).
Период замера: 200 мс.

### Xiao Sense турель (192.168.88.23) — sketch_mar16a.ino

| Компонент | Пин |
|-----------|-----|
| Серво Tilt | **D2 (GPIO 2)** |
| Серво Pan | **D3 (GPIO 4)** |
| Камера OV2640 | Встроенная (expansion board) |

Серво: 50Hz PWM, 500-2400μs. Полный ход: 0°..180° (обе оси).
UDP задача на Core 0, серво write() на Core 1 (через dirty-флаги).

## WiFi

| Устройство | IP | Примечание |
|---|---|---|
| Ровер (XIAO ESP32S3) | 192.168.88.24 | Нужен резерв DHCP в роутере! |
| Xiao Sense (Pan/Tilt + Camera) | 192.168.88.23 | Нужен резерв DHCP в роутере! |
| Сеть | Rhanet | |

**Важно:** обе прошивки используют DHCP (нет `WiFi.config()`).
Роутер должен резервировать IP по MAC-адресу, иначе адреса могут смениться.
